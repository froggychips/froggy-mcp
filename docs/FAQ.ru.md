# froggy-mcp — FAQ

Быстрые ответы для пользователей Froggy, которые хотят подключить его к Claude Code.

---

## Что делает froggy-mcp?

Это мост. Claude Code умеет вызывать инструменты, которые работают на твоей машине. Этот мост даёт Claude Code доступ к двум возможностям Froggy:

- **`froggy_context`** — запрашивает у Froggy текущий снимок экрана (OCR-текст, после того как Redactor вырезал секреты)
- **`froggy_transcript`** — запрашивает у Froggy последнюю расшифровку аудио

Поток: Claude Code вызывает инструмент → мост пересылает запрос Froggy через локальный Unix-сокет → Froggy отвечает → Claude видит содержимое экрана.

---

## Почему отдельный репозиторий, а не часть Froggy?

Froggy — самостоятельный демон, работающий без Claude Code. MCP-мост нужен только тем, кто использует Claude Code. Разделение даёт:

- Можно запускать Froggy без Claude Code (управление памятью + локальная LLM работают как обычно)
- Пользователи Claude Code могут добавить мост, не скачивая исходники всего Froggy
- У моста ноль внешних Swift-зависимостей — только stdlib + Foundation

---

## Экран уходит к Anthropic?

Сам мост ничего никуда не отправляет. Это локальный форвардер: читает из Unix-сокета (Froggy) и пишет в stdio (Claude Code).

Что происходит дальше — задача Claude Code: он упаковывает результат инструмента и отправляет в API Anthropic как часть разговора. Так что да, OCR-текст экрана доходит до серверов Anthropic — так же как любое твоё сообщение Клоду.

Если не хочешь чтобы содержимое экрана уходило к Anthropic: не вызывай `froggy_context`, или ограничь через `allowedTools` в `.claude/settings.json`.

---

## Как установить?

Сначала должен быть запущен демон Froggy.

```bash
git clone https://github.com/froggychips/froggy-mcp
cd froggy-mcp
swift build -c release
```

Добавь в `.mcp.json` проекта (или в `~/.claude.json` для глобального доступа):
```json
{
  "mcpServers": {
    "froggy": {
      "command": "/path/to/froggy-mcp/.build/release/froggy-mcp"
    }
  }
}
```

## Работает без Froggy?

Процесс моста запустится, но вызовы инструментов завершатся ошибкой соединения — на сокете никто не слушает. Сначала запусти демон Froggy.

---

## В чём разница между двумя инструментами?

| Инструмент | Возвращает | Требует |
|---|---|---|
| `froggy_context` | OCR-текст экрана (последний снимок, секреты вырезаны) | Разрешение на запись экрана |
| `froggy_transcript` | Последняя расшифровка аудио | Разрешение на микрофон |

Оба обрабатываются на устройстве внутри Froggy ещё до того как мост их увидит. Мост только пересылает результат.

---

## Это замедлит Claude Code?

Мост — тонкий форвардер stdio↔сокет. Накладные расходы пренебрежимо малы. OCR-обработка происходит в фоновом цикле Froggy вне зависимости от того подключён мост или нет — дополнительных затрат на вызов инструмента нет, Froggy уже сделал работу заранее.

---

## Можно использовать только один инструмент?

Да. Используй `allowedTools` в `.claude/settings.json`:

```json
{
  "allowedTools": ["mcp__froggy__froggy_context"]
}
```

Это уберёт `froggy_transcript` из списка доступных инструментов Claude в этом проекте.
